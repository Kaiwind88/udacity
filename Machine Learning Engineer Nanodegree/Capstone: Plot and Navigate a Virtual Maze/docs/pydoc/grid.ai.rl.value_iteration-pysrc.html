  9      ''' 
 10      Value iteration is a method of computing an optimal MDP policy and its value. Value  
 11      iteration starts at the "end" and then works backward, refining an estimate of either  
 12      Q* or V*. There is really no end, so it uses an arbitrary end point.  
 13      Let Vk be the value function assuming there are k stages to go, and let Qk be the Q-function  
 14      assuming there are k stages to go. These can be defined recursively. Value iteration starts with  
 15      an arbitrary function V0 and uses the following equations to get the functions for k+1 stages to  
 16      go from the functions for k stages to go: 
 17   
 18   
 19      Qk+1(s,a) = sigma-s' P(s' | s,a) (R(s,a,s') + gamma * Vk(s')) for k >= 0 
 20       
 21      Vk(s) = maxa Qk(s,a) for k > 0 
 22       
 23      It can either save the V[S] array or the Q[S,A] array. Saving the V array results in less storage,  
 24      but it is more difficult to determine an optimal action, and one more iteration is needed to determine  
 25      which action results in the greatest value.  
 26       
 27      Here Living Reward is 0.0, Robot Noise is 0.0 and Goal State Reward is 1.0. Discount (gamma) is set to  
 28      0.9, and num of iterations (k) is set to 100. The transition probability for a state (cell location)  
 29      is based on the number of wall openings in that state. 
 30      '''
