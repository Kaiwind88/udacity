 11      ''' 
 12      In Q-learning and related algorithms, an agent tries to learn the optimal policy from  
 13      its history of interaction with the environment. A history of an agent is a  
 14      sequence of state-action-rewards: 
 15       
 16      <s0,a0,r1,s1,a1,r2,s2,a2,r3,s3,a3,r4,s4...>, 
 17   
 18      which means that the agent was in state s0 and did action a0, which resulted in it receiving  
 19      reward r1 and being in state s1; then it did action a1, received reward r2, and ended up in  
 20      state s2; then it did action a2, received reward r3, and ended up in state s3; and so on.  
 21      Q*(s,a), where a is an action and s is a state, is the expected value (cumulative discounted  
 22      reward) of doing a in state s and then following the optimal policy. Q-learning uses temporal  
 23      differences to estimate the value of Q*(s,a). In Q-learning, the agent maintains a table of Q[S,A],  
 24      where S is the set of states and A is the set of actions. Q[s,a] represents its current estimate  
 25      of Q*(s,a). An experience <s,a,r,s'> provides one data point for the value of Q(s,a). The data point  
 26      is that the agent received the future value of r + gamma * V(s'), where V(s') = maxa' Q(s',a'); this is the  
 27      actual current reward plus the discounted estimated future value. This new data point is called  
 28      a return. The agent can use the temporal difference equation to update its estimate for Q(s,a): 
 29       
 30      Q[s,a] <- Q[s,a] + alpha(r+ gamma * maxa' Q[s',a'] - Q[s,a]) or, equivalently,  
 31      Q[s,a] <- (1-alpha) Q[s,a] + alpha(r+ gamma * maxa' Q[s',a']) 
 32       
 33      Q-learning learns an optimal policy no matter which policy the agent is actually following (i.e.,  
 34      which action a it selects for any state s) as long as there is no bound on the number of times it  
 35      tries an action in any state (i.e., it does not always do the same subset of actions in a state).  
 36      Because it learns an optimal policy no matter which policy it is carrying out, it is called an off-policy method. 
 37   
 38      Here total num of learning episodes is set to 100. Learning rate, alpha=0.5, Discount, gamma=0.9 and  
 39      Exploration, epsilon=0.3. Both state transition function and reward function are missing. Agent learns  
 40      both by interacting with the environment during its learning episodes. 
 41      '''
